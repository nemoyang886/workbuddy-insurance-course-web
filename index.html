06 / 44
正确使用 Agent

先让 Agent 理解你，再让它帮你工作

一句话原则
不要把 Agent 当成临时聊天框，而是把它当成能理解你、读取资料、按流程交付的工作助手。
先给背景
告诉它你是谁、服务或处理什么对象、常用什么表达、哪些边界不能碰。
再给规则
告诉它每次怎么读取资料、怎么判断问题、怎么输出结果。
认识我
身份画像、对象类型、表达风格和工作边界，是 Agent 做事前的上下文。
知道怎么做
工作职责、默认流程、资料读取顺序和输出格式，是 Agent 的执行规则。
会自己检查
事实与判断分开，隐私先匿名，表达符合你的风格，交付前先自检。

08 / 44
使用边界

第一次用 Agent，先讲清楚三件事

01

你是谁

你的身份、目标、当前任务、专业方向或学习方向、表达风格。

02

它可以看什么

个人介绍、内容样本、匿名案例、学习资料、知识库文件。

03

它不能碰什么

姓名、电话、身份证、保单号、银行卡、未经脱敏的健康信息。

只要涉及客户、家人、朋友、学习资料或工作资料，都要先想清楚哪些能给 AI 看，哪些不能给。资料越清楚、权限越克制，Agent 越容易做出可用结果。

资料准备

每位学员准备四类资料

个人定位
经历、专业方向、服务人群、价值主张
表达样本
3-5条朋友圈、一篇长文、常用沟通表达
目标客户
年龄段、职业、家庭阶段、高频问题
表达边界
不用的词、避免的营销方式、偏好风格
资料准备检查表
01先写清楚我是谁、服务谁、擅长什么。
02再放入自己真实说话方式和内容样本。
03客户或对象资料先匿名化，只保留必要背景。
04提前说明哪些表达不要用，哪些边界不能碰。
原则：少量、清晰、可验证。资料越干净，Agent 越容易稳定输出。

工作规则
20 / 44
交付流程

表达、合规与交付流程

交付流程提示词：保险顾问版 + 通用版
【保险顾问版】
表达风格：专业、自然、清晰，避免空泛口号、过度煽情、制造焦虑，像真实微信或朋友圈表达。

保险业务边界：不承诺收益，不作医疗、法律或税务结论，不虚构客户案例，不使用客户真实敏感信息，不替客户做决定。

默认交付流程：
1. 先判断这是内容、定联、资料整理还是复盘任务
2. 再分清事实、判断、建议和待确认问题
3. 给出内容角度或执行计划
4. 等我确认后生成正式版本
5. 同时给出“可直接用版本”和“需要我确认的地方”
6. 交付前完成合规与风格自检

---

【通用版】
表达风格：自然、清楚、具体，避免空泛口号、夸张表达、模板感，像我本人在真实沟通。

工作边界：不虚构经历和数据，不替别人做决定，不处理未经脱敏的他人隐私，不把建议说成确定结论。

默认交付流程：
1. 先判断这是内容、学习整理、资料整理、沟通还是复盘任务
2. 再分清事实、判断、建议和待确认问题
3. 给出内容角度或执行计划
4. 等我确认后生成正式版本
5. 同时给出“可直接用版本”和“需要我确认的地方”
6. 交付前完成清晰度、可执行性和风格自检
